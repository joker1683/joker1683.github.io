function duplicates(arr) {
    var result = [];
    arr.forEach(function(elem) {
        if(arr.indexOf(elem) !== arr.lastIndexOf(elem) && result.indexOf(elem) === -1) {
            result.push(elem)
        }
    })
    return result;
}

Http协议

简述http协议：
浏览器向服务器发送请求，服务器向浏览器返回html文本，基于的就是http协议,http基于TCP协议，http分为请求和响应两部分，请求又分为请求行、请求头和请求体，响应分为响应行、响应头和响应体
http协议是无状态的，
http包括持久连接和非持久连接：
非持久连接：每次传输都需要单独建立一次TCP连接
持久连接：只需建立一次TCP连接，客户端和服务器可一直请求和响应
TCP/IP 三次握手:客户端发出SYN包加一个序号m，服务端收到之后将收到的序号加一并发送ACK(m+1)＋SYN包带一个新产生的序号n一同返回给客户端，客户端收到之后将序号加一并返回ACK(n+1)包
四次挥手:主动方发出FIN包带一个序号m，关闭主动方的连接，被动方收到之后发送ACK包带序号m+1，被动方再次发送FIN包并带一个序号,关闭被动方的连接，主动方收到后发送ACK包并将收到的序号加一
网络七层模型:物理层 数据链路层 网络层 传输层 会话层 表示层 应用层
TCP四层模型:数据链路层 网络层 传输层 应用层
缓存:强制缓存 协商缓存 查看缓存相关头信息cache-control expires看是否满足强制缓存，如果不满足携带缓存相关头信息请求服务器，服务器查看之后是告知直接从缓存读取还是返回最新的资源内容。
http2和http1.1的区别:
1.1 的头信息是文本格式的，2的头信息和数据都是二进制的。
2可以复用TCP连接，如果一个请求的数据量过大，可以先把已处理的数据先返回给客户端，然后暂时挂起这个请求，先接收别的请求,最后再回头处理这个耗时的请求
2可以给每个数据包一个标识，数据包可以不按顺序发送，取消发送数据的时候可以发送一个信号，而不用关掉TCP连接
2可以让服务器主动推送给客户端资源，例如一些静态资源
2头信息用gzip进行压缩，并且客户端和服务端共同维护一张头信息表，所有字段都会生成一个索引号，以后就不用 发送同样的字段了
http1.1 和 http1.0的区别:
1.1可以复用TCP连接，以加快传输速度
http有几种请求方法:
GET POST PUT DELETE OPTIONS HEAD CONNECT TRACE 一共八种
https协议的过程
https协议由两部分组成 一是http 二是ssl/tls 也就是在http协议基础上又加了一层处理加密信息的模块，客户端和服务端传输的数据都是经过TLS加密过的。
过程：
发送客户端支持的加密协议及版本，ssl、tls
服务端从中筛选合适的加密协议
服务器端返回证书，证书中含有公钥
客户端使用根证书验证证书合法性
客户端生成对称密钥，通过证书中的公钥加密，发送到服务器端
服务器端使用私钥解密，获取对称密钥，使用对称密钥加密数据
证书可以使用自己生成，也可以向专门的https证书提供商进行购买。这两种的区别是自己生成的证书是不被浏览器信任的,所以当访问的时候回提示不安全的网站，需要点击信任之后才能继续访问
因为浏览器中预置了一些https证书提供商的证书，在浏览器获取到服务器的https证书进行验证的时候就知道这个https证书是可信的；而自己生成的证书，浏览器获取到之后无法进行验证是否可信，所以就给出不安全的提示。
非对称加密算法，非对称加密有一个秘钥对，叫做公钥和私钥，私钥自己持有，公钥可以公开的发送给使用的人。使用公钥进行加密的信息，只有和其配对的私钥可以解开。
